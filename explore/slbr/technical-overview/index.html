Internationalized URL Rendering & Homograph Protection

SLBr displays internationalized domain names in a readable format instead of a "xn--" representation.

This improved readability for users, allowing domains that contain emojis, unicode characters, or symbols to appear as intended, rather than as strings that are difficult to read.

However, rendering Unicode domains directly introduced a significant security concern, that is homograph attacks.

Homograph attacks exploit different look alike characters to create deceptive domain names, such as replacing Latin characters with Cyrillic or Greek.

A few examples include:

  1. vvikipedia.com
  2. аррӏе.com

To address this risk, SLBr implemented a homograph detection and highlighting system within the omnibox.

When potentially confusable or suspicious characters are detected, they are highlighted in orange, making them distinct from other characters.

This approach introduces the following benefits:

  1. Internationalized domains remain readable.
  2. Suspicious domains are highlighted.
  3. Users are not forced to interpret raw Punycode.

However, the initial implementation rendered every single character of the domain as an individual Run.

This was inefficient and introduced unnecessary rendering overhead.

SLBr now uses a new rendering approach:

  1. Characters are grouped into segments.
  2. Each segment is rendered as a single text run.
  3. Suspicious and normal characters are batched separately.

This significantly reduced the number of elements created, improving performance.
